1. The Prompt Formula

~5 minutes
The Idea

A good prompt has four parts — Role, Context, Task, Format. Adding these turns vague output into useful output.

Why It Matters

Most people get bad AI output because they give bad briefs. The formula fixes 80% of that in 30 seconds.

Try It Yourself

Paste each into the Try Out section, ChatGPT, or Claude. Compare the two outputs side by side.

🧪 Open Try Out section →
Vague prompt
Write an email to my customers about our sale.
Briefed prompt
Role: You are a friendly local restaurant owner. Context: We're doing 20% off this weekend for our 5-year anniversary. Customers are regulars — families and couples for special occasions. Task: Write the announcement email. Format: Warm and personal, under 100 words, one clear call-to-action.

Once you see how much the formula helps, the next problem shows up — you don't want to retype your role and tone every single time. That's what memory solves.

2. Stop Repeating Yourself

~5 minutes
The Idea

Modern AI tools let you set who you are and how you like things written — once. Every chat after that already knows.

Why It Matters

Re-explaining yourself in every chat is the biggest hidden time-waster in AI. Set it once, save hours every week.

Try It Yourself

Open ChatGPT or Claude. Paste this once into a new chat. Then start a fresh chat and ask anything — notice how AI already knows who you are.

Memory seed prompt
Please remember the following about me for all future conversations: - I am a [your role, e.g., school teacher / small business owner / parent] - I prefer responses that are [warm and direct / concise and practical / detailed and thorough] - When I ask you to write something, default to [under 200 words / no jargon / a friendly conversational tone] - I value [practical advice / honesty over hype / step-by-step clarity]
💡 You only run this once. After that, every chat starts with this context already loaded. To check it worked: open a new chat and ask "What do you remember about me?"

Memory works for you personally. But what about a project where multiple AI tools or teammates are involved? That needs a shared rulebook.

3. Project Rulebooks and Reusable Workflows

~10 minutes
The Idea

When you move beyond personal use into projects, two new pieces help: AGENTS.md and Skills. Both are simple text files that travel with your project.

Why It Matters

Memory is personal. AGENTS.md and Skills are how context travels — across tools, across teammates, across projects.

Part A AGENTS.md

Think of it as a README for AI. A single Markdown file that tells any AI tool how to work on your project. Used by 60,000+ projects on GitHub. Stewarded by the Linux Foundation as an open standard.

Try It Yourself

Copy this template, fill in the blanks for any project you work on, and save it as AGENTS.md in your project folder.

AGENTS.md template
# My Project AGENTS.md ## How to work on this project - Always run tests before finishing: [your test command] - Code style: simple, readable, comments only when needed - Never edit files in [folder name] — that's the old system ## Gotchas - [The thing that has tripped up every newcomer]
Part B Skills

A folder with a SKILL.md file that packages a repeatable workflow. Anthropic introduced these in October 2025. Use the same Skill across Claude.ai, Claude Code, and the API.

Try It Yourself

Paste this into Claude. It becomes a reusable Skill you can trigger any time you need to write in your brand voice.

Brand voice Skill
--- name: my-brand-voice description: Use when writing customer emails or social posts for my restaurant. Applies our voice — warm, family-oriented, never pushy or marketer-y. --- # How we write to our customers ## Voice - Talk like a neighbor, not a marketer - Use first names when possible - Never say "valued customer" or "exclusive offer" ## Structure 1. One-line warm opener 2. The actual point 3. Soft call-to-action 4. Sign off with the owner's name

~5 minutes
The Idea

Prompts, memory, AGENTS.md, and Skills each have their own home in a well-organized project.

Why It Matters

When all four work together, AI stops feeling like a clever toy and starts feeling like a teammate.

Try It Yourself

5. Context Engineering — Going Deeper Advanced · Optional

Optional · ~10 min

If you're already comfortable with prompts, memory, and Skills — this is the next idea worth understanding.

Most people think the secret to better AI is a better prompt. The people who actually build with AI for a living think differently. They treat context — everything the AI sees before it answers — as a system they design, not a sentence they type. It has a name now: context engineering.

"Prompt engineering was the 2023–2024 phase. Reliability comes from the context system, not from clever wording."
What "context" really means

Context is everything inside the AI's view at the moment you ask. It's not just your latest message. It includes:

  • ⚙️The system instructions that shape the AI's role and tone
  • 💬Your conversation history with this AI in this session
  • 📄Files, documents, or images you've shared
  • 🧠Memory and saved preferences you've set up
  • 🔍Tool outputs, search results, or retrieved data
💡 The takeaway: Better AI is rarely about smarter prompts. It's almost always about cleaner, more relevant context.
The four moves: write, select, compress, isolate

LangChain — one of the teams building agent infrastructure for serious companies — describes context engineering as four core moves. They map neatly to how you should think about every AI interaction.

✍️ Write
The idea: Save useful context outside the conversation so it survives between sessions.
In practice: Memory features, Skills, AGENTS.md, project notes.
Why it matters: Without writing, every chat starts from zero.
🎯 Select
The idea: Pull only the relevant context into the AI's view at the right moment.
In practice: Choose which files to share, which Skill to apply, which past thread to reference.
Why it matters: AI can't filter what's useful. You have to.
🗜️ Compress
The idea: Summarize or trim context so the most important information stays visible.
In practice: Ask AI to "summarize what we've decided so far" before moving on.
Why it matters: Long context degrades AI performance — more on this below.
🔒 Isolate
The idea: Use separate AI sessions for separate tasks.
In practice: Don't draft an email, debug a problem, and plan a vacation in one chat. Open three.
Why it matters: AI doesn't compartmentalize. Mixing topics confuses output.
Why AI gets worse the longer you chat

You may have noticed: after 20–30 back-and-forth messages, AI starts getting things wrong. Forgetting earlier instructions. Repeating itself. Losing the thread. This isn't your imagination. Researchers have a name for it: context rot.

In 2025, researchers at Chroma tested 18 leading AI models. Every single one degrades as context grows — even models with million-token windows. Anthropic's own research has shown that context above 100,000 tokens reduces reasoning quality.
📍 Lost in the Middle — Researchers at Stanford documented that AI pays the most attention to information at the very beginning or very end of context. Anything stuck in the middle is more likely to be missed — even if it's important.

Three habits to fight context rot:

  • 1Start a new chat for a new task. Don't reuse a long thread.
  • 2Put critical instructions at the start or end of your message — never bury them in the middle.
  • 3Periodically ask AI to "summarize what we've covered so far" and start a fresh chat with that summary.
From clever prompts to designed context

The shift in how serious teams use AI is best described this way: stop treating the prompt as a magic spell. Start treating context as infrastructure.

Test yourself — three questions

Try answering before you reveal. The point isn't to gatekeep — it's a quick way to check the ideas landed.

1. You've been chatting with AI for 30 messages and the answers are getting worse. What's the best move?
Start a new chat. Ask AI to summarize the key decisions from the old one, then paste that summary into the fresh conversation. You get a clean context without losing anything important.
2. You're sharing a 50-page document with AI to ask one specific question. Where in your message should you put that question?
At the very end — after the document. AI pays the most attention to the start and end of context. The middle is where information goes to be missed.
3. You ask AI to draft an email, then debug your spreadsheet, then plan a vacation — all in the same chat. What's wrong?
You're mixing contexts. Each task bleeds into the others. Open three separate chats — one per task — and the quality of all three goes up.
